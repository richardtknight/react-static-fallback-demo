# A simple React app with a pre-rendered static HTML fallback

This react app is running React 18 on Vite and utilising react-snap to pre-render the content on the different routes into plain html files within the dist folder.

If javascript is enabled as normal the React app will Hydrate as expected, however if it's disabled instead the app will fall back to the static HTML files.

This means that when we view the source code rather than seeing the usual React output:
```
<!doctype html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <link rel="icon" type="image/svg+xml" href="/vite.svg" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Vite + React + TS</title>
    <script type="module" crossorigin src="/assets/index-B5oBCM6M.js"></script>
    <link rel="stylesheet" crossorigin href="/assets/index-fQS_wizX.css">
  </head>
  <body>
    <div id="root"></div>
  </body>
</html>
```
